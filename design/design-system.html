Design Principles

iOS 17+ Native Patterns

  • SF Pro font system with Dynamic Type support
  • Semantic colors that adapt to Light/Dark mode automatically
  • 8pt grid system for consistent spacing and rhythm
  • 44pt minimum touch targets (Apple HIG compliance)
  • Native iOS components: TabView, NavigationStack, Form, List
  • Spring animations with iOS-native curves (0.3s-0.5s duration)
  • Pull-to-refresh gesture on scrollable views
  • Haptic feedback on key interactions

Accessibility First

  • VoiceOver labels on all interactive elements
  • Dynamic Type support for text scaling
  • High contrast mode compatibility
  • Color-blind friendly status indicators (icons + color)
  • Keyboard navigation support on iPadOS
  • Reduce Motion support for animations

Performance Targets

  • 60fps animations on iPhone 12 and newer
  • Launch time under 400ms
  • API response rendering under 100ms
  • Smooth scrolling with lazy loading for capsule grid
  • Optimized for iOS 17+ (no backward compatibility needed)

Implementation Notes for Mobile Dev

Color References: Use existing Colors.swift definitions. All colors are semantic and adapt to Light/Dark mode automatically.

Typography: Use existing Typography.swift definitions. All fonts support Dynamic Type for accessibility.

Spacing: Use existing Spacing.swift constants. All layouts follow 8pt grid for visual consistency.

Components: Build using SwiftUI native components (Button, Card from existing design system). Reference Theme.swift for button styles.

Animations: Use springNormal for most interactions, springFast for quick responses, springBouncy for celebration effects.

Assets: Use SF Symbols for all icons. No custom icon assets needed - SF Symbols are native, accessible, and adapt to Dynamic Type.
